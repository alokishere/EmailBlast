1. Core Privacy Promise

MailBlast is designed to work in session-only mode. During login and sending, required data is processed in server RAM for the active session.

2. No Persistent Personal Data Storage

  • We do not permanently store personal account data in an application database.
  • Session data is kept temporarily in RAM during active use.
  • On logout, session data is destroyed from server memory.

3. Session Lifecycle

  • Session starts after successful Google login.
  • Session is used only to keep you authenticated and send emails from your account.
  • Session is removed on logout or automatic session expiry.

4. Data Use Purpose

Data available during the active session is used only to authenticate the user and execute outgoing email send requests initiated by that user.

5. Policy Changes

If these privacy rules change in the future, this page will be updated before new behavior is applied.
